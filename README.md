# meh

> *"What if we made an AI write an AI coding assistant? How bad could it be?"*

A terminal-based AI coding assistant written in Rust. The entire thing — every line of code, every architectural decision, every test — was generated by Claude Opus 4.6 in a series of conversations where a human said "implement next step" roughly 44 times and went to make coffee.

This is not a product. This is a science experiment wearing a trench coat pretending to be a product.

## What Is This

**meh** is a Ratatui-based TUI that talks to LLMs (Anthropic, OpenAI, Gemini, OpenRouter), executes tool calls, manages conversations, and does all the things a coding assistant should do — except it was written by an AI that learned Rust from reading the internet, so buyer beware.

## The Experiment

In early 2026, we wanted to answer a simple question:

**Can a frontier coding model generate a non-trivial, multi-module Rust application from an architectural spec — with working async concurrency, a TUI, streaming HTTP, tool execution, and permission systems — in one continuous session?**

The answer is: *kind of, mostly, with caveats, and it's funnier than you'd expect.*

### What Actually Happened

1. Human wrote a detailed architecture doc (CLAUDE.md) and 37 step-by-step implementation plans
2. Claude Opus 4.6 implemented each step, one at a time, generating code + tests
3. An external model (Gemini 3.1 Pro) audited the output and found real bugs
4. Opus fixed the bugs that Gemini found
5. The cycle repeated until the humans ran out of things to nitpick
6. The UI has a settings panel that sometimes ghost-renders because ratatui's double-buffer is spicy

## Features (That Theoretically Work)

- **4 LLM providers** — Anthropic, OpenAI, Gemini, OpenRouter with streaming SSE
- **Tool system** — read/write files, execute commands, search, apply patches
- **Permission system** — Ask, Auto, YOLO modes (the YOLO mode was the AI's idea)
- **MCP client** — stdio and SSE transports for Model Context Protocol servers
- **Sub-agents** — delegate tasks to child agents (what could go wrong)
- **TUI** — async Ratatui with crossterm EventStream, settings panel, slash commands
- **Task history** — save/resume conversations with debounced auto-save
- **Token counting** — tiktoken-rs with context window tracking
- **Cost tracking** — per-model pricing with color-coded display
- **Config hot-reload** — file watcher with debounce
- **Graceful cancellation** — Ctrl+C cancels, double Ctrl+C force-quits
- **`.mehignore`** — gitignore-syntax path protection for secrets
- **`.mehrules`** — user rules with YAML frontmatter conditionals
- **jiter JSON parsing** — proper incremental parser for streaming tool args
- **mimalloc** — because even AI-generated code deserves a fast allocator

## Installation

```bash
git clone https://github.com/delfianto/meh.git
cd meh
cargo build --release
```

You'll need API keys. Set them as environment variables or use `/settings` in the TUI:

```bash
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...
export GEMINI_API_KEY=AI...
```

Then:

```bash
./target/release/meh
# or with a prompt:
./target/release/meh "fix the authentication bug"
# or in YOLO mode (no approval prompts, pray):
./target/release/meh --yolo "refactor everything"
```

## Architecture

The architecture is actually pretty reasonable (the AI read a lot of Claude Code before writing this):

```
┌─────────────────────────────────────────────────┐
│                  meh (binary)                   │
├─────────────────────────────────────────────────┤
│  TUI (Ratatui)  ◄──►  Controller  ◄──►  State  │
│       ▲                     │                   │
│       │               ┌─────┴─────┐             │
│  EventStream          ▼           ▼             │
│  + select!      TaskAgent    ToolExecutor       │
│                     │         + Permissions     │
│                 Provider      + IgnoreCtrl      │
│                (streaming)                      │
└─────────────────────────────────────────────────┘
```

- **Async-first** — Tokio runtime, channels everywhere, the TUI is event-driven
- **Permission-gated** — Every tool call checks `PermissionController` before execution
- **Bounded channels** — The UI channel has backpressure (learned this the hard way)
- **mimalloc** — Global allocator for better multi-threaded `String` allocation performance
- **jiter** — Pydantic's JSON parser for incremental streaming tool arguments

For the gory details, see `plans/ARCHITECTURE.md` and the 44 step files in `plans/`.

## The Audit Trail

This project was audited twice by Gemini 3.1 Pro, which found:

1. **Permission system was completely bypassed** — tools executed immediately without checking the permission controller. Fixed.
2. **AutoSaver had a starvation bug** — `tokio::select!` recreated the sleep timer every loop iteration, so it never fired during streaming. Fixed.
3. **All channels were unbounded** — potential OOM under fast streaming. UI channel now bounded.
4. **Tool executor returned "not yet implemented" for everything** — the actual executor existed but wasn't wired in. Fixed.
5. **Regex-based JSON parser** — only captured strings, broke on escaped quotes. Replaced with jiter.
6. **TUI busy-polled at 60fps** — even when idle. Replaced with async EventStream.
7. **Agent tasks leaked on cancel** — JoinHandle wasn't stored. Now aborts after 5s timeout.

Every bug was real. Every fix was generated by Opus. The circle of AI life.
